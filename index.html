Design a Cucumber/Selenium2/Rspec Test Suite against a Source-Ambiguous Web Application that is both Powerful and Stakeholder-Friendly
Cucumber means
  • behavior-driven development
  • Gherkin
  • acceptance testing, regression testing
  • Business/Business Analyst/Client/Stakeholder-readable
Selenium 2 means
  • automating browsers
  • WebDriver
  • taking screenshots and recording video
RSpec means
  • behavior-driven development
  • powerful expectations
Source-Ambiguous means
  • legacy applications
  • no cucumber/rails integration
  • no rspec/rails integration
  • no database manipulation/seeding
"Powerful and Stakeholder-Friendly" means
  • readable tests
  • executable specs
  • tagged tests
  • proof of success-or-failure
